# Page by Page
'Page by Page'는 읽은 책을 기록하고, 공유하는 습관을 기르는 웹사이트 플랫폼이다. 기본에 많은 앱들이 책 기록과 관리 기능을 제공하고 있지만, 그 경험을 다른 사람들과 나누며 소통하는 온라인 커뮤니티의 필요성을 깨달았다. 책을 기록하는 것에 그치지 않고, 사용자들이 책에 대한 다양한 의견을 나누고, 독서에 대한 습관을 기를 수 있는 공간을 제공하는 웹사이트이다.

## 주요 기능
- 캘린더: 캘린더에 읽은 책들을 등록하며, 한눈에 사용자의 읽은 양을 파악할 수 있다. 또한 책의 기억에 남는 구절이나, 책의 내용을 기록하도록 한다
- 베스트셀러: 게시판에 가장 언급이 많았던 책들을 메인페이지에 나타내어, 사용자들이 가장 관심이 많은 책이 무엇인지 보여준다
- 게시판: 추천할만한 책, 혹은 책에 대한 토론을 이루는 게시판을 책의 카테고리별로 나누어 커뮤니티를 형성하게 한다
  
<br>

## 프로젝트 소개

###  1.0 목표
- 개인 독서 통계를 간단하고 직관적으로 확인
- 독서 기록과 공유를 통해 독서 습관을 촉진
- 선호 장르와 독서 패턴 분석 기반 맞춤형 책 추천
- 캘린더 및 리스트로 읽은 책 관리, 자동화된 도서 정보 입력

---

### 1.1 회원 관리 기능
- **회원가입**:
    - 사용자는 아이디, 비밀번호, 이메일, **프로필** 사진을 입력.
    - 비밀번호는 해시 암호화로 저장.
    - **입력값 검증** (중복 아이디, 이메일 등)
- **로그인**:
    - 아이디와 비밀번호로 인증.
    - **성공 시 세션 생성**, 메인 페이지로 리다이렉트
- **로그아웃**:
    - 세션 삭제 후 메인 페이지로 이동

---

### 1.2 메인 페이지
-  **게시판 입장**:
  - 카테고리별로 대표 책을 띄우고 양쪽 스크롤을 통해 게시판 입장 가능
  - 책 위에 마우스를 올리면, 게시판 입장 버튼 등장
- **캘린더 입장**
- **책 추천**

---

### 1.3 게시판 기능
- 장르 6가지로 게시판 카테고리 구분: 소설/문학, 심리/철학, 사회/현대 이슈, 경제/경영, 과학/기술, 예술/문화
- 게시물 작성: 독서플랫폼에 맞게 웹사이트의 정체성을 유지할 수 있도록, 게시글은 책 정보와 책 추천여부 등을 필수 입력
- 책 추천여부: 게시글 작성시 사용자가 추천할 책인지, 아닌지 여부 설정
- **책 정보 API**: 책 제목을 입력하면 자동으로 책 저자, 내용 등 정보 제공

---

### 1.4 게시물 보기
- **게시글 관리**:
    - 글 작성, 수정, 삭제 가능
    - 제목, 내용, 작성자, 작성일 포함
- **댓글 관리**:
    - 게시글에 댓글 작성, 수정, 삭제 가능
    - 최신 댓글이 위로 올라오는 구조

---

### 1.5 게시물 리스트
- 6가지의 카테고리별로 게시글 분류 후 리스트하여 표시
- **페이지네이션**:
    - 게시글 목록을 10개씩 페이지로 나눠 표시

---

### 1.6 독서캘린더
- **일정 관리**:
    - 날짜를 선택해 책을 입력하면 캘린더에 책 버튼 생성
    - 버튼 클릭 시 책 정보와 메모 작성 화면 표시
    - **추천여부에 따라 책 이름 옆에 색을 표시**(예.추천시🟢  /비추천시:🔴 )캘린더에 표시함
- **책 정보 API:**
    - Open Library API 또는 Google Books API를 활용하여 입력된 책의 정보를 자동으로 표시.
    - **책 이미지를 가져옴.**

---

### 1.7 **마이페이지와 관리자 기능**
> 좌. 마이페이지/ 우. 관리자기능 페이지
- **프로필 정보**:
    - 사용자 정보 조회 및 수정 가능.
- **(독서 통계)**: 추후 개발예정
    - 이번 달에 읽은 책 수, 선호 장르 비율 등을 시각화.
    - Chart.js 또는 Google Charts API 활용.
- **사용자 관리**:
    - 사용자 목록 확인, 권한 관리.

## 어려웠던 점
시간에 쫓기다보니 디자인에 신경쓰지 못하고, 최대한 단순하게 하려고 했다. 완성하고 보니 디자인의 통일성이 부족하고 아쉬운 점도 있었지만, 개인 프로젝트로 처음 이 정도 수준까지 구현한 점에는 만족한다. 점점 하다보니 시간상 뺏지만 나중에 추가하면 좋을 것 같은 기능이 있어 남겨본다.
1. 인기책 순위를 api가져와서 메인에 리스트로 보여주기
2. 게시판페이지에 검색기능 추가하기
3. 도서검색기능 혹은 yes24나 알라딘 사이트와 연동해놓기
4. 메인에 tree라이브러리 사용해보기
5. 책 추천여부와 별개로 별점 선택 고려하기
6. 즐겨찾기 기능 추가하기
7. 사용자의 독서통계를 달, 주별로 그래프로 표시하기
